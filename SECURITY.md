# Security Policy

## Reporting a Vulnerability

We take the security of Clawrap seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Please do NOT create a public GitHub issue for security vulnerabilities.**

Instead, report it by:

1. **Email**: Send an email to [security@YOUR_EMAIL_DOMAIN] (or use GitHub's private vulnerability reporting if enabled)
2. **GitHub Private Vulnerability Reporting**: Go to Security → Report a vulnerability

### What to Include

Please provide as much information as possible:

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact
- Any possible mitigations you've identified

### Response Timeline

- **Within 48 hours**: We will acknowledge receipt of your report
- **Within 7 days**: We will provide an initial assessment and planned next steps
- **Ongoing**: We will keep you informed of progress toward a fix

### Disclosure Policy

- Please allow us reasonable time to fix the issue before public disclosure
- We will work with you to coordinate disclosure timing
- We appreciate responsible disclosure and will credit researchers (with permission)

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | ✅ |
| Development (main branch) | ✅ |
| Older versions | ❌ |

We recommend always using the latest released version for security updates.

## Security Best Practices for Users

- Keep Clawrap updated to the latest version
- Never share your API keys or configuration files
- Verify downloads match official releases
- Report suspicious behavior immediately
