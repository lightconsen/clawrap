# Contributing to Clawrap

Thank you for your interest in contributing to Clawrap! This document provides guidelines and instructions for contributing.

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md) to help keep our community welcoming and inclusive.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check existing issues as you might find the problem has already been reported. When creating a bug report, please include:

- **A clear, descriptive title**
- **Steps to reproduce** the issue
- **Expected behavior** vs **actual behavior**
- **Your environment**: OS, Node.js version, Clawrap version
- **Screenshots or logs** if applicable

Use the bug report template when creating an issue.

### Suggesting Features

Feature suggestions are always welcome! Please provide:

- **A clear, descriptive title**
- **Detailed description** of the proposed feature
- **Use case**: Explain why this feature would be useful
- **Possible implementation** ideas (optional)

### Pull Requests

1. Fork the repository
2. Create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Test your changes locally
5. Commit with clear, descriptive messages following [Conventional Commits](https://www.conventionalcommits.org/)
6. Push to your fork and submit a PR

### Development Setup

See [docs/install.md](docs/install.md) for development environment setup instructions.

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting (Prettier/ESLint if configured)
- Write meaningful commit messages
- Add comments for complex logic

## Pull Request Guidelines

- Fill out the PR template completely
- Keep PRs focused - one feature/fix per PR
- Reference related issues (e.g., "Closes #123")
- Ensure all tests pass (if applicable)
- Update documentation if needed

## Questions?

Feel free to open an issue with the "question" label for any questions about contributing.
